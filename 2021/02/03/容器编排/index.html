6. 编排中遇到的问题

  1. docker 中运行的容器服务不可以使用后台启动， docker 发现不了后台启动的服务, 现象会是刚启动docker 服务后立马停止。
  2. docker 内容器的通信需要创建的时候注意下： 这里uwsgi的配置文件中socket = :8000 是只可以容器间的端口可见。socket ==> http 的时候，容器间就不可见，但外部就可以进行直接的访问。
  3. uwsgi 中的属性 daemonize 是指将 uwsgi 后台运行。如果指定了这个配置项目后，现象就如1中所示。
